# Game Design Canon

## Core loop

У каждой игры должен быть короткий понятный цикл:

```text
появилась цель -> ребёнок смотрит -> игра показывает прогресс -> действие засчитано -> награда -> следующая цель
```

Если core loop нельзя объяснить одной строкой, игра слишком сложна для первого релиза.

## Правила хорошей gaze-игры

- Одна основная задача за раз.
- Крупные интерактивные элементы.
- Явное отличие цели от фона.
- Мгновенная обратная связь при входе взгляда.
- Плавный dwell progress.
- Мягкая ошибка без резкого наказания.
- Пауза всегда доступна взрослому.
- Нет обязательных быстрых реакций в базовом уровне.
- Нет мерцания, перегруза, случайных вспышек.
- Звук поддерживает действие, но не пугает.

## Flow

Игра должна поддерживать баланс:

- слишком легко - ребёнку скучно;
- слишком сложно - ребёнок теряет контроль;
- оптимально - ребёнок видит успех и хочет повторить.

## Feedback ladder

Обратная связь должна идти слоями:

1. Hover/gaze enter: цель оживает.
2. Dwell progress: видно, что выбор идёт.
3. Success: мягкий звук, анимация, изменение мира.
4. Progress: шаг сессии обновился.
5. Reward: сад вырос, пузырь лопнул, слово собрано.

## Ошибки

Ошибка должна быть информацией, не поражением. Лучше говорить через игру:

- `Попробуем ещё раз`.
- `Посмотри сюда`.
- `Давай медленнее`.
- `Осталось чуть-чуть`.

## Polishing

Полировка не равна большему количеству эффектов. Для этих игр polish означает:

- ясность;
- спокойствие;
- отзывчивость;
- предсказуемость;
- ощущение успеха;
- красоту без перегруза.
