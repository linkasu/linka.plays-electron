## Назначение

Спокойная партия в домино против простого бота: ребёнок выбирает костяшку из руки, сравнивает её числа с открытыми концами цепочки и ставит подходящую костяшку слева или справа.

Самостоятельная формулировка для меню: Ставь костяшки к таким же числам и собери цепочку домино.

## Игровой цикл

```text
новая партия -> стартовая костяшка -> ход игрока -> ход бота -> взять из базара, если хода нет -> победа/итог
```

## Правила

- Полный набор double-six: 28 костяшек от `0:0` до `6:6`.
- `gentle` раздаёт 4 костяшки игроку и боту, `standard` — 5, `challenge` — 6.
- Открытые числа показываются слева и справа от цепочки.
- Костяшка подходит, если одно из её чисел совпадает с открытым числом на конце цепочки.
- Если костяшка подходит к двум концам, игра показывает два больших выбора: `Слева` и `Справа`.
- После успешного хода игрока бот делает один спокойный ход: ставит подходящую костяшку или берёт одну костяшку из базара.
- Если у игрока нет подходящего хода, активируется кнопка `Взять`.
- Победа игрока наступает, когда рука игрока пуста. Партия также завершается при пустом базаре и отсутствии ходов.

## Управление взглядом

- Основной ввод должен быть gaze-first; mouse fallback сохраняется для отладки и занятий без трекера.
- Минимальный целевой размер из registry: 148 px.
- Базовый dwell из registry: 1300 ms.
- Последний Electron CDP audit нужно обновить после перехода на полноценную партию с ботом.

## Дефектологическая ценность

- тренирует планирование и выбор действия
- переводит настольную/аркадную идею в спокойный пошаговый формат
- тренирует очередность хода и перенос правила на новую ситуацию

## Метрики и сессия

- Сессия должна использовать общий game session flow и завершаться через результат для взрослого.
- Важные метрики: успешные выборы, ошибки/отмены, dwell time, подсказки, потеря валидного взгляда.
- Рекомендуемая длительность занятия: 180 сек.

## QA checklist

- Route открывается в Electron, не только в обычном браузере.
- На 800×600 и 1024×600 основные цели видны без ручного скролла.
- HUD, prompt и overlays не перекрывают активные цели.
- Для canvas/fullscreen игры PNG подтверждает непустую сцену и читаемые active zones.
- Звук, если есть, остаётся опциональным и не ломает gameplay при ошибке загрузки.
- Для игр с правилами model tests покрывают правильный ответ, ошибки и завершение раунда.
