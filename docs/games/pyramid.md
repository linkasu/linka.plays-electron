# Пирамидка

## Unity source

- Scene: `Assets/Games/Pyramid/Pyramid.unity`.
- Asset: `Assets/Resources/Games/Games/Pyramid.asset`.
- Script: `TorusController.cs`.

## Core mechanic

Собрать пирамидку в правильном порядке. Ребёнок выбирает кольца взглядом, игра укладывает их на стержень.

## Игровой цикл

```text
кольца лежат -> выбрать следующее -> кольцо переместилось -> следующий шаг -> пирамидка собрана
```

## Управление взглядом

- Dwell target на кольцах.
- На easy подсвечивать правильное кольцо.
- На standard показывать правило: от большого к маленькому.

## Дефектологическая ценность

- Последовательность.
- Сравнение размера.
- Планирование.
- Понимание порядка.

## Современное переосмысление

Не завязываться на имена объектов `0..3`, как в Unity. Генерировать модель колец явно: размер, цвет, правильный индекс.

Правила выбора вынесены в `src/frontend/games/pyramid/model.ts`: модель создаёт 4 кольца, определяет следующее ожидаемое кольцо, возвращает размещённые кольца в порядке установки и фиксирует исход выбора. Ошибка порядка не блокирует сборку: выбранное кольцо всё равно ставится, а сессия записывает ошибку.

## Audio/TTS

- Игра включает тихий звук по умолчанию.
- Стартовый TTS объясняет задачу без раскрытия следующего правильного кольца.
- Верный выбор, ошибка и завершение сопровождаются короткими TTS-фразами и мягкими piano-feedback сигналами.
- Выбор кольца заблокирован, пока звучит TTS/feedback, чтобы не накладывать ответы.
- Если в сборке были ошибки порядка, перед итоговым окном появляется короткая cutscene-разбор: сравнение выбранного порядка с правильным и спокойный разбор сложных шагов.

## Уровни сложности

- Gentle: 3 кольца, правильное подсвечено.
- Standard: 4-5 колец, порядок по размеру.
- Challenge: порядок по цветовой подсказке или смешанные признаки.

## Метрики

- Правильные шаги.
- Ошибки порядка.
- Время выбора каждого кольца.
- Подсказки.

## QA checklist

- Нельзя выбрать уже уложенное кольцо.
- Ошибка не ломает последовательность.
- Hit area кольца достаточно крупная.
- TTS не подсказывает конкретное правильное кольцо.
- При ошибках финальная cutscene появляется до итогового диалога и объясняет правило без обвиняющего тона.
