# Теннис

## Unity source

- Scene: `Assets/Games/TableTennis/TableTennis.unity`.

- Scripts: `BallController.cs`, `PlayerController.cs`.

## Core mechanic

Ракетка следует по вертикали за взглядом. Нужно отбивать мяч. Это игра continuous control, поэтому она должна быть поздним этапом после базовых игр.

## Игровой цикл

```text
мяч летит -> ребёнок ведёт ракетку взглядом -> отбил -> rally продолжается -> короткий итог
```

## Управление взглядом

- Continuous vertical control.
- Gaze y-coordinate управляет ракеткой.
- X-coordinate игнорируется или используется минимально.
- Сглаживание обязательно.
- Easy режим должен иметь slow ball и большую ракетку.

## Дефектологическая ценность

- Непрерывный контроль.
- Слежение за движущимся объектом.
- Предикция движения.
- Координация взгляда и результата.

## Современное переосмысление

Не делать соревновательный Pong на выбывание. Сделать кооперативный `держим мяч в игре`. Цель - серия успешных касаний, а не победа над AI.

## Уровни сложности

- Gentle: большая ракетка, мяч, soft wall assists.
- Standard: обычная ракетка, умеренная скорость.
- Challenge: меньше assist, быстрее мяч, длиннее rally.

## Метрики

- Rally length.
- Успешные отбивания.
- Промахи.
- Средняя дистанция ракетки до мяча.
- Потери gaze во время движения.

## MVP

- Canvas Pong-like сцена.
- Ракетка следует за gaze y.
- Мяч отскакивает.
- Session по времени, не по поражениям.

## Therapy-ready

- Настройка скорости.
- Настройка размера ракетки.
- Assist mode.
- Отчёт по continuous control.

## Polished

- Дружелюбная тема: воздушный шарик, пляжный мяч, пинг-понг.
- Кооперативный персонаж-партнёр.
- Награда за серию.

## QA checklist

- Ракетка не дёргается от gaze noise.
- Мяч не ускоряется бесконечно.
- Промах не вызывает резкий game over.
