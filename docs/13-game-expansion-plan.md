# Game Expansion Plan

## Цель

Расширять LINKa plays как лестницу сложности, а не как случайный набор мини-игр.

Порядок развития:

1. Ambient / no-fail игры без ошибки и давления.
2. Первые крупные цели взглядом.
3. Поиск и внимание.
4. Сборка и последовательности.
5. Речь, AAC и выбор.
6. Счёт, формы и логика.
7. Классика и стратегия.
8. Непрерывное и адаптивное управление.

Новые игры должны соблюдать канон проекта: короткий core loop, мягкая награда, отсутствие наказания, крупные цели, optional sound, no autoplay dependency, адаптация dwell/target/motion/options.

## Общие Правила Для Новых Игр

| Правило | Требование |
|---|---|
| Ошибка | Ошибка не является поражением: повтор, подсказка, замедление или уменьшение вариантов. |
| Звук | Только optional, короткий, тихий, не continuous background по умолчанию. Ошибки аудио не ломают игру. |
| Dwell | Gentle: 1200-1600 мс; standard: 900-1200 мс; challenge: 600-900 мс. |
| Цели | Static target от 120 px, answer card от 180 px, moving effective zone от 140 px. |
| Сессия | 60-180 секунд по умолчанию, длиннее только для стратегии/классики. |
| Управление | Не смешивать сложное управление и сложную когнитивную задачу в одной early игре. |
| Адаптация | Менять targetScale, dwellMs, motionSpeed, число вариантов, distractors, hints. |
| UI | Vuetify-first для карточек/меню, scoped CSS только для игровых сцен. |

## Особые Требования К Классике

| Игра | Безопасная gaze-адаптация |
|---|---|
| Тетрис спокойный | Нет быстрой реакции в gentle. Сначала выбор колонки, затем поворот, затем мягкое подтверждение. Ghost piece всегда виден. |
| Тетрис adaptive | Gravity растёт только при стабильном gaze. Есть pause after lock, undo на 1 ход, no hard game over в gentle. |
| Змейка спокойная | Не стрелки на мгновенную реакцию, а waypoint/zones: змейка идёт к выбранной зоне. |
| Змейка adaptive | Safe walls, крупная еда, низкая скорость, сглаженные повороты, автозамедление перед едой. |

## Параллельная Разработка

Параллельные агенты не должны работать в одном рабочем дереве. Для каждой игры нужен отдельный branch и отдельный git worktree.

Формат ветки:

```text
game/<slug>
```

Формат worktree:

```text
/Users/aacidov/linka/worktrees/linka-plays-<slug>
```

Минимальный состав каждой игровой ветки:

1. `src/frontend/games/<slug>/<PascalName>Game.vue`
2. `src/frontend/games/<slug>/model.ts` если нужна логика раунда.
3. `src/frontend/games/<slug>/model.test.ts` если есть чистая модель.
4. Optional `audio.ts`, только тихие короткие optional cues.
5. Обновление `src/frontend/data/games.ts`.
6. Обновление `src/frontend/router/index.ts`.
7. Проверка `npm run typecheck` и релевантных unit tests.

Первая параллельная партия должна быть небольшой: 4-6 игр. Иначе merge conflicts в `games.ts` и `router/index.ts` станут дороже самой разработки.

Рекомендуемая первая партия:

| Ветка | Игра | Почему можно параллельно |
|---|---|---|
| `game/firefly-meadow` | Светлячковая поляна | isolated ambient canvas |
| `game/quiet-bubbles` | Тихие пузыри | isolated ambient canvas |
| `game/big-button` | Большая кнопка | isolated simple dwell component |
| `game/yes-no` | Да / нет | isolated AAC choice game |
| `game/find-color` | Найди цвет | isolated visual search model |
| `game/tower` | Башня | isolated sequencing choice game |

После первой партии нужен интеграционный merge branch, где вручную сводятся `games.ts` и `router/index.ts`, затем запускаются `typecheck`, `test:unit`, `build`, Electron UI smoke.
