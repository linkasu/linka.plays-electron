# Gaze Engine

## Задача

Gaze engine должен сделать Tobii и мышь одинаковыми источниками pointer-событий для игр. Игра не должна знать, пришёл ввод от Tobii или от мыши, кроме случаев отображения статуса.

## Pointer model

Минимальная модель pointer:

```ts
type GazePointer = {
  x: number;
  y: number;
  valid: boolean;
  source: "tobii" | "mouse";
  timestamp: number;
};
```

## Основные проблемы gaze input

- Шум координат.
- Невалидные точки.
- Дрожание на границе цели.
- Случайный выбор при простом взгляде, Midas touch.
- Усталость при долгом dwell.
- Сложность маленьких целей.
- Разная точность у разных детей и позиций головы.

## Dwell click

Любая кликабельная цель взглядом должна иметь dwell progress:

- `enter` - взгляд вошёл в цель.
- `stay` - взгляд удерживается.
- `progress` - заполнение индикатора.
- `cancel` - взгляд ушёл до завершения.
- `click` - dwell завершён.

Начальные значения:

- easy: `1200-1600ms`.
- medium: `900-1200ms`.
- hard: `600-900ms`.

Для детей с тяжёлыми моторными нарушениями default должен быть мягким: лучше дольше, но стабильнее.

## Anti-Midas-touch

Чтобы ребёнок не выбирал случайно, использовать:

- dwell progress;
- визуальное подтверждение;
- cancel при уходе взгляда;
- опциональное second-step подтверждение для сложных игр;
- крупные безопасные зоны;
- cooldown после выбора.

## Target magnet

Для некоторых игр gaze pointer можно мягко притягивать к центру активной цели. Это снижает дрожание и делает опыт управляемым. Магнит не должен менять сырые метрики: в лог пишется и raw, и effective позиция.

## Smooth pursuit

Для игр `Бабочки`, `Рыбки`, `Теннис` полезен режим плавного слежения. Там важно не только dwell, но и траектория:

- насколько долго взгляд рядом с движущимся объектом;
- насколько плавно ребёнок следует;
- как часто теряется valid gaze;
- не слишком ли быстро движется цель.

## Invalid gaze

Если gaze invalid:

- не прыгать в `0,0`;
- не засчитывать выбор;
- показывать мягкий статус `Потеряли взгляд`;
- сохранять последнюю valid точку только как визуальный ghost, но не как активный click;
- после восстановления дать короткий grace period.

## Calibration flow

Перед серией занятий нужен быстрый экран проверки:

- статус Tobii;
- виден ли взгляд;
- точность по 5-9 точкам;
- рекомендации: ближе, дальше, поправить посадку, перекалибровать.

## Метрики gaze engine

- `validGazeRatio`.
- `meanFixationMs`.
- `targetEntries`.
- `targetCancels`.
- `dwellSuccesses`.
- `rawPathLength`.
- `effectivePathLength`.
- `lostGazeEvents`.
