# Политика обезличенной аналитики

В packaged LINKa plays обезличенная аналитика работает только после явного включения пользователем. На первом экране до выбора приложение показывает варианты «Включить аналитику» и «Не отправлять». Настройку можно изменить позже на том же экране. Постоянная ссылка: <https://plays-metric.nkolinka.ru/privacy>.

## Состояния privacy

- `Unknown` - решение ещё не принято. Telemetry runtime не создаётся: приложение не регистрирует installation, не открывает и не сканирует spool, не отправляет очередь и не записывает `app_started`. При первом запуске `0.1.18` каталог `telemetry-v1`, оставшийся от версии без opt-in, удаляется целиком до показа окна.
- `Enabled` - разрешён сбор перечисленных ниже обезличенных событий. Новая app session начинается в момент включения, а не при запуске приложения до согласия.
- `Disabled` - сбор и отправка прекращены. Локальный `telemetry-v1`, включая spool и installation identity, удаляется.

Privacy transitions выполняются последовательно. При quit или update-restart main process сначала закрывает IPC gate для новых решений, ожидает уже принятую transition и только затем завершает telemetry lifecycle.

Выбор хранится main process в `userData/privacy-preferences.json` и доступен renderer только через typed preload IPC. Старый informational notice `linka-metrics-privacy-version` из renderer `localStorage` не читается и согласием не считается. Учётных записей и activation-flow в приложении нет.

## Что собирается

- после включения - случайный installation ID, не связанный с учётной записью или аппаратным ID;
- версия и build приложения, платформа, версия ОС и locale;
- app/game session UUID, route name и game ID из закрытого allowlist без query/search, режим меню и категория игры;
- числовая настройка dwell;
- lifecycle приложения, игры, Tobii и updater;
- агрегаты игровой сессии: активная длительность, pause, шаги, успехи, ошибки, подсказки, отмены, потери взгляда, изменения сложности и раздельные sample counts;
- fingerprint ошибки без исходного текста.

## Что запрещено

- имя, контакты и другие данные пользователя;
- текст, фразы, слова, expected/actual ответы;
- непрерывные или отдельные координаты взгляда и указателя, timestamp pointer sample;
- `targetId`, board/FEN и другие снимки игрового состояния;
- пути файлов, error message и stack;
- raw Tobii diagnostics.

Raw diagnostics остаётся только локальной функцией отчёта и копирования в буфер. Клиент не содержит IPC/API для её удалённой загрузки. Отключение аналитики не влияет на Tobii, mouse fallback, игры и offline-режим.

## Хранение

Сервер хранит уже принятые данные бессрочно, без автоматического TTL. Удаление по запросу не предусмотрено. При `Enabled` до доставки записи находятся в bounded spool приложения; сеть никогда не блокирует игру. Доставка at-least-once, поэтому backend должен считать `event_id` идентификатором повторной записи. Переход в `Disabled` удаляет только локальные данные и не может отозвать уже принятые сервером события.
