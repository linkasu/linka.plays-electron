# Game Session

## Задача

Game session заменяет Unity `GameCounter`. Это единая модель прогресса, ошибок, паузы, результата и событий.

## Session lifecycle

```text
idle -> starting -> running -> paused -> running -> finishing -> finished
```

## Общие поля

```ts
type GameSession = {
  gameId: string;
  startedAt: number;
  finishedAt?: number;
  status: "idle" | "running" | "paused" | "finished";
  step: number;
  maxSteps: number;
  score: number;
  mistakes: number;
  hintsUsed: number;
  settings: SessionSettings;
};
```

## Общие действия

- `startSession`.
- `pauseSession`.
- `resumeSession`.
- `finishSession`.
- `recordSuccess`.
- `recordMistake`.
- `recordHint`.
- `recordGazeEvent`.

## Что считается успехом

Успех зависит от игры:

- выбран правильный объект;
- удержан взгляд нужное время;
- завершён элемент последовательности;
- пойман движущийся объект;
- введена правильная буква или число;
- ребёнок проследил за объектом заданное время.

## Ошибки

Ошибки не должны быть жёстким наказанием. Ошибка фиксируется для метрик, но игра должна помогать:

- повторить вопрос;
- подсветить правильную область;
- уменьшить количество вариантов;
- увеличить dwell;
- замедлить объект;
- предложить паузу.

## Завершение

Сессия завершается по одному из условий:

- достигнут `maxSteps`;
- истекло время;
- взрослый завершил вручную;
- ребёнок устал, и взрослый нажал паузу/стоп;
- игра достигла финального состояния.

## Экран результата

Для ребёнка:

- короткая похвала;
- визуальный итог;
- кнопки `Ещё раз` и `В меню`.

Для взрослого:

- длительность;
- успешные попытки;
- ошибки;
- подсказки;
- valid gaze ratio;
- средний dwell до выбора;
- комментарий системы: `слишком много отмен dwell`, `часто терялся взгляд`, `можно уменьшить dwell`.
