# UI UX Guidelines

## Общий стиль

UI строится на Vuetify: карточки, кнопки, чипы, диалоги, layout primitives и theme tokens. Custom CSS допускается только для игровых canvas/svg областей и точной локальной визуальной настройки.

## Главное меню

Меню должно помогать выбрать игру по навыку, а не только по названию.

Карточка игры показывает:

- название;
- короткое описание;
- категорию;
- тренируемые навыки;
- сложность;
- статус готовности;
- рекомендуемую длительность.

## Перед запуском игры

Нужен экран настроек:

- профиль сложности;
- длительность;
- dwell time;
- размер целей;
- включить/выключить звук;
- уровень подсказок;
- режим высокой контрастности;
- источник ввода и статус Tobii.

## Внутри игры

HUD должен быть минимальным:

- `В меню`;
- `Пауза`;
- статус Tobii;
- прогресс;
- возможно очки/ошибки, если это не отвлекает.

Для ребёнка важнее игровой мир, чем счётчики.

## Доступность

- Крупные кликабельные зоны.
- Высокий контраст опционально.
- фон.
- Без мелкого текста внутри игровой области.
- Без hover-only информации.
- Без быстрых исчезающих сообщений.
- Звук можно отключить.
- Инструкции повторяемые.

## Mobile/Desktop

Electron обычно desktop, но layout должен не ломаться на разных окнах. Canvas-игры обязаны корректно resize.
